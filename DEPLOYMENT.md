# Production Deployment & Vercel Release Guide

This document maps out the precise sequence required to compile, validate, and launch the **Compulsion Portfolio** live on Vercel under your custom subdomain: **`praneeth-portfolio.vercel.app`**.

---

## 🛠️ Step 1: Pre-Flight Production Build Check

Before pushing to production, verify that the compilation bundle builds cleanly with zero TypeScript errors or path resolution mismatches.

Run this command in your local project terminal:
```bash
npm run build
```

* **Expected Outcome**: A compiled `/dist` output directory containing optimized, compressed chunks (< 300KB initial chunk load).
* **If it fails**: Paste the compilation logs into the agent chat for immediate remediation.

---

## 🚀 Step 2: Push to Git Repository (GitHub)

Vercel utilizes event-driven Git hooks to automatically deploy new commits pushed to your repository.

1. Stage all changes (including the new `vercel.json` rewrite file):
   ```bash
   git add .
   ```
2. Commit the codebase:
   ```bash
   git commit -m "feat: rebrand to indigo precision and optimize animations"
   ```
3. Push to your main branch:
   ```bash
   git push origin main
   ```

---

## 🌐 Step 3: Deploy on Vercel (Free Hobby Plan)

1. Log into your account at **[Vercel.com](https://vercel.com)** (sign up using your GitHub account for automatic integration).
2. From the Vercel Dashboard, click **Add New...** and select **Project**.
3. Locate your portfolio repository and click **Import**.
4. **Crucial Configuration step for custom link:**
   * Under the **Project Name** input field, change the default name to exactly: **`praneeth-portfolio`**
   * This guarantees that your free default domain generated by Vercel resolves precisely to:
     ```
     https://praneeth-portfolio.vercel.app
     ```
5. Leave the remaining build settings at default:
   * **Framework Preset**: `Vite` (Auto-detected)
   * **Build Command**: `npm run build`
   * **Output Directory**: `dist`
6. Click **Deploy**. Vercel will compile and host the app in under 60 seconds!

---

## 🧭 Step 4: Verification Checklist

Once the deployment finishes, verify these three markers of a successful launch:
1. **Direct Routing Check**: Navigate to `https://praneeth-portfolio.vercel.app/projects/market-research-sugar-cosmetics` and refresh the page. (Our new `vercel.json` should intercept the request and load the case study detail page correctly instead of showing a 404).
2. **Animation check**: Confirm that the snappier name reveal scramble effect triggers beautifully.
3. **Automatic Redeployments**: Any future changes you push to your GitHub `main` branch will automatically trigger a background production rebuild on Vercel within seconds.
